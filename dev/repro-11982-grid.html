Single icon-only menu-bar item inside a narrow grid action column

Realistic host instead of an isolated div: each row below is its own small vaadin-grid with a flex-grow="0" action column, styled like the "action column" pattern from vaadin/docs#2309, rendering a single icon-only menu-bar item.

The exact pixel width where this collapses is font/icon-rendering-dependent — it differs between macOS, Windows and headless Linux Chromium, so a hardcoded width tuned on one machine does not reliably reproduce on another. This page instead self-calibrates on load: it renders one throwaway, unconstrained menu-bar to measure this exact button's needed column width on your browser right now, then builds 4 action columns a fraction of a pixel narrower than that — a real, sub-pixel overflow the item should never collapse over.

Note: which of the 4 rows show the bug, and at which zoom level, varies by machine — the same font-rendering differences that make a hardcoded width unreliable also mean the exact collapse boundary shifts relative to any calibration. On unpatched code, one or more rows may already show the bug at 100% zoom with no zooming needed at all; on others it may only appear once you zoom in or out a few percent. There is no single width/zoom combination guaranteed to reproduce it identically on every OS — that inconsistency is itself the bug (integer-rounding two independently-rounded values means the false-collapse boundary depends on exactly where the sub-pixel geometry happens to land, which differs by renderer).

To reproduce manually: click Measure first, at whatever zoom the page loaded with. If any row is already outlined in red, that is the bug on unpatched code. If none are, zoom the actual browser page — Ctrl/Cmd +/-, Ctrl/Cmd+scroll, or on Windows, Settings → System → Display → Scale — a step at a time in either direction, clicking Measure after each step, until one flips. A collapsed row shows its single item nested inside its own overflow ellipsis (click it: it opens a submenu whose only item is the same ellipsis button). On fixed code, none of the 4 rows should ever collapse, at any zoom. Each row is a separate 2-column grid, not one grid with many columns — cramming many fixed columns into a single grid row was tried first and shifted the danger band unpredictably narrow; separate grids reproduce the same layout math this bug actually depends on.
